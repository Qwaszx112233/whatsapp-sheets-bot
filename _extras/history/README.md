# History archive index

This folder keeps documentation that is useful for audit trail, migration context, or postmortem reference, but is **not** part of the active operational doc set.

## Typical contents
- stage reports
- stabilization notes
- canonicalization audits
- self-containment audits
- delivery notes
- patch notes
- import helper notes from earlier packaging stages

## Archive policy
- keep active operational docs in the repository root
- move one-off historical notes here
- do not treat any file here as the canonical current procedure unless the active root docs explicitly tell you to

## Reading order for current work
1. `README.md`
2. `ARCHITECTURE.md`
3. `RUNBOOK.md`
4. `SECURITY.md`
5. `CHANGELOG.md`

Use the files in this folder only when you need historical context.
