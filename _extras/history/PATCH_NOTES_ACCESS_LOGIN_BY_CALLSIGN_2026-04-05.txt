ACCESS LOGIN BY CALLSIGN — 2026-04-05
=====================================

Что изменено
------------
1. Вход для обычного пользователя переписан с ручного копирования hash на сценарий:
   - система определяет текущий user-key сама;
   - если ключ не привязан, показывается кнопка "Увійти";
   - пользователь выбирает только позывной;
   - роль берётся только из ACCESS на сервере.

2. AccessControl.gs
   - добавлен новый столбец ACCESS: self_bind_allowed
   - добавлены функции:
     * listBindableCallsigns()
     * bindCurrentKeyToCallsign(callsign)
   - regular describe() больше не обязан отдавать полный hash;
   - debug-режим может отдать полный hash только когда это явно запрошено.

3. Stage7MaintenanceApi.gs
   - добавлены API:
     * apiStage7ListBindableCallsigns
     * apiStage7BindCurrentKeyToCallsign
   - apiStage7GetAccessDescriptor теперь использует безопасный режим без полного hash
   - apiStage7DebugAccess использует debug-режим с полным hash

4. Js.Security.html
   - полностью переписан клиентский access-flow
   - добавлен modal входа по позывному
   - access banner теперь умеет показывать кнопку "Увійти"
   - убран основной пользовательский сценарий "скопируй hash и вставь руками"
   - debug-кнопки полного hash оставлены только как технический режим

5. Js.Api.html
   - добавлены клиентские методы:
     * MaintenanceApi.listBindableCallsigns()
     * MaintenanceApi.bindCurrentKeyToCallsign(callsign)

6. Js.Helpers.html
   - access debug log теперь использует masked hash, а не пустой currentKey

Важно по ACCESS
----------------
Новый столбец:
- self_bind_allowed

Если столбца ещё нет, схема ACCESS будет автоматически расширена.

Рекомендуемая логика значений:
- TRUE  -> запись можно показывать в окне "Увійти"
- FALSE -> запись нельзя самопривязывать

Если ячейка пустая, действует fallback по роли:
- viewer / operator / maintainer -> self-bind разрешён
- admin / sysadmin / owner -> self-bind запрещён
