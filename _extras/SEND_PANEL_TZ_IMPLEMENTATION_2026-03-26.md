# SEND_PANEL — внедрение ТЗ от 2026-03-26

## Что изменено

Реализована новая модель последних трёх колонок SEND_PANEL без изменения логики формирования `A3:D40`:

- `E = Status`
- `F = Sent`
- `G = Action`

## Что сохранено

- старая логика заполнения `A3:D40`
- старые источники данных SEND_PANEL
- общая бизнес-логика проекта
- работа через существующие server/use-case/sidebar слои

## Что изменено по логике

### 1. Новый контракт листа
Заголовки приведены к виду:

- `A2 = FIO`
- `B2 = Phone`
- `C2 = Code`
- `D2 = Tasks`
- `E2 = Status`
- `F2 = Sent`
- `G2 = Action`

### 2. Status
Колонка `E` теперь содержит только:

- `✔` — строка допущена к отправке
- `✘` — строка не допущена к отправке

### 3. Sent
Колонка `F` теперь содержит только:

- `✔` — строка зафиксирована как отправленная
- `✘` — строка ещё не зафиксирована

### 4. Action
Колонка `G` содержит готовую action-ссылку.
Sidebar читает её напрямую из листа и не генерирует заново.

### 5. Sidebar
Старый сценарий `open -> pending -> confirm` убран из основного потока.
Теперь:

1. sidebar открывает ссылку из `G`
2. сразу после открытия вызывает серверную фиксацию
3. строка получает `F = ✔`
4. UI сразу обновляется как отправленный

### 6. Серийная отправка
Кнопка массовой отправки в sidebar теперь по каждой строке:

- открывает сохранённую action-ссылку
- сразу ставит `Sent = ✔`
- обновляет состояние без ручного подтверждения
